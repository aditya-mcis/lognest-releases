Features

What's included

All the power of LogNest, now with language-aware UI built on industry-standard i18n tooling.

🌐

Auto language detection

Automatically picks up your browser language on first launch. Falls back to English when a locale isn't available.

💾

Preference persisted

Language choice is saved to localStorage under lognest_lang. Survives page reloads and restarts.

⚡

Zero runtime cost

All translations are bundled at build time via react-i18next. No network requests for locale files.

🔌

Works with all integrations

Backend API, Python client, Node.js client, Java client, curl — unchanged. Only the UI is translated.

🎯

Interpolation support

Dynamic values like log counts, tier names, and thresholds are correctly interpolated in every language.

🧩

Easy to extend

Add a new language by dropping a single JSON file in src/i18n/locales/ and registering it in index.js.

Get started

Set up in 3 steps

Clone the i18n branch, install, and run. Your existing backend and log clients need no changes.

1

Clone the repo

git clone https://github.com/Yuli-Developer/lognest-i18n.git

2

Install and start frontend

cd lognest-i18n/frontend && npm install && npm run dev — opens on localhost:3000

3

Start the backend

Run the Spring Boot backend from the main LogNest repo on localhost:9000. Frontend connects automatically.

Run the test suite:
bash test.sh

Checks backend health, frontend reachability, all 6 locale files for completeness, source file existence, and a full production build.
